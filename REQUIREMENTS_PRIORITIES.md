# FastShort 需求文档（整合版）

本文档用于明确二次开发需求、优先级与验收口径，确保用户全流程可用。

## 1. 目标与范围
目标：用户从注册登录、浏览短剧、观看播放、充值/解锁、互动评论形成完整闭环。

范围：
- 端用户应用（H5/移动端）
- 管理后台
- 后端 API

## 2. P0 必须完成（上线阻断）
### 2.1 用户全流程可用
- 注册/登录/鉴权稳定可用
- 首页与分类可正常展示
- 分集播放可用（无黑屏）
- 充值/支付闭环可用

### 2.2 支付能力
- 订单创建
- 回调校验与幂等
- 支付记录与对账
- 失败重试/补单

### 2.3 存储与播放稳定性
- OSS/CDN 替换本地存储
- 视频/封面可稳定访问
- 播放失败重试或兜底

### 2.4 内容审核与风控
- 上下架与审核流程
- 举报与屏蔽
- 管理员操作审计

### 2.5 外部内容源接入（uuuka API）
- 定时/手动同步内容
- 去重与缓存
- 仅同步元数据（默认不直链播放）

## 3. P1 重要增强（业务核心能力）
- 会员/订阅体系（权益、时长、分集解锁）
- 推荐与运营位配置（首页推荐、置顶、类目推荐）
- 数据报表（播放、转化、留存、付费）
- 评论/互动完善（审核、举报、屏蔽）
- 多语言与多地区（内容、货币、支付渠道）

## 4. P2 体验优化
- 分享与拉新（微信/微博/Twitter）
- 新手引导与首播体验优化
- 客服与反馈系统
- 性能优化（首屏、列表、播放）

## 5. 交付与验收口径（核心）
- 登录/注册成功率 > 99%
- H5 端可顺利播放分集（视频可访问）
- 支付订单与回调闭环可用
- 管理后台可完成：分类/剧集/分集的新增与上架
- 外部内容源可同步入库并在后台可见
